# JavaScript Events - Complete Notes

---

# 1. What is an Event?

## Fundamental Concept

An **event** is a signal generated by the browser whenever something interesting happens.

Examples:

* User clicks a button
* User types in an input field
* Page loads
* Image finishes loading
* Network connection changes
* Video starts playing

Think of JavaScript as an **event-driven language**.

Instead of continuously checking:

```javascript
Did user click?
Did user click?
Did user click?
```

JavaScript says:

> "Tell me when a click happens, and I'll execute my code."

This makes applications efficient and interactive.

---

# 2. Event-Driven Programming

Traditional Programming:

```text
Start
↓
Execute Line 1
↓
Execute Line 2
↓
Execute Line 3
↓
End
```

Event-Driven Programming:

```text
Page Loads
↓
Wait for Events
↓
User Clicks
↓
Run Click Handler
↓
Wait Again
```

The browser spends most of its time waiting for events.

---

# 3. Event Lifecycle

When an event occurs:

```text
User Action
↓
Browser Creates Event Object
↓
Browser Dispatches Event
↓
Event Travels Through DOM
↓
Registered Handlers Execute
```

Example:

```javascript
button.addEventListener("click", handler)
```

When clicked:

```text
Click Occurs
↓
Browser Creates MouseEvent
↓
Handler Executes
```

---

# 4. Why Do We Need Events?

Without events:

* Buttons wouldn't work
* Forms couldn't be submitted
* Games couldn't respond to keyboard input
* Videos couldn't be controlled
* Interactive websites wouldn't exist

Events are the foundation of web interactivity.

## `this` in Event Handlers & Memory Considerations

In DOM event listeners, `this` inside a regular function handler refers to the element the listener is bound to. Examples and gotchas:

```html
<button id="b">Click</button>
```

```js
const btn = document.getElementById('b');
btn.addEventListener('click', function (e) {
	console.log(this === btn); // true
	console.log(e.currentTarget === btn); // also true
});

// Arrow functions inherit `this` from the outer scope
btn.addEventListener('click', (e) => {
	console.log(this); // not the button, depends on surrounding scope
});
```

Memory notes:

- Storing DOM nodes in long-lived closures (e.g., modules or caches) can prevent them from being garbage-collected even after removal from the document.
- Always remove listeners when elements are removed or when no longer needed: `element.removeEventListener(...)`.

Prefer using `e.currentTarget` when you need a reliable reference to the element that received the event (works regardless of binding style), and avoid arrow-functions for handlers that rely on `this` being the element.

# 7. Event Propagation

---

## Capturing Phase

```text
Document
↓
HTML
↓
BODY
↓
DIV
↓
BUTTON
```

Top → Bottom

---

## Target Phase

```text
BUTTON
```

Actual target reached.

---

## Bubbling Phase

```text
BUTTON
↑
DIV
↑
BODY
↑
HTML
↑
DOCUMENT
```

Bottom → Top

# 8. Event Delegation

Instead of:

```javascript
100 buttons
100 listeners
```

Use:

```javascript
Parent Listener
```

and inspect:

```javascript
e.target
```

Benefits:

* Less memory
* Better performance
* Easier maintenance
* Works with dynamic elements
